Algunos sitios optimizados para dispositivos móviles están mal configurados de manera que no muestran a los buscadores la información que se busca. Por ejemplo, a los usuarios de móviles se les muestra una página de error o se le redirecciona a una página irrelevante al acceder a una URL pero a los usuarios de escritorio se les muestra el contenido buscado. Alguno de estos problemas, detectados por el robot de google se muestran en esta nueva pestaña y pueden dañar significantemente su sitio web y la experiencia del usuario y son la base de algunos de los recientes cambios anunciados por Google en los rankings de búsqueda en las búsquedas realizadas por teléfonos móviles.

Actualmente se puede usar el apartado de errores de rastreo para identificar de manera concreta páginas en tu sitio que tienen este tipo de problema.



Algunos de los errores incluidos son los siguientes:
  • Errores del servidor: Un error del servidor es cuando el robot de Google obtiene un error HTTP al rastrear una página
  • No encontrado o 404 leve: Una página puede mostrar un mensaje de "no encontrado" a el bot de google pero sin lanzar un error de HTTP 404, cuando eta página se detecta es un 404 leve
  • Redirección defectuosa: Una redirección defectusa es un error especifico de dispositivos móviles que ocurre cuando una URL de escritorio normal y corriente redirige a un contenido no relevante para su búsqueda. Un ejemplo básico es cuando todas las URLs del sitio web de escritorio redirigen a la home del sitio web para móviles en lugar de la URL específica.
  • URLs bloqueadas: Una url bloqueada es cuando el archivo robots.txt explícitamente indica que no se puede rastrear la url por el robot de Google para smartphpones. Normalmente, las directivas solamente para smarthpones son erroneas, en este caso deberías de investigar la información de configuración de tu servidor.

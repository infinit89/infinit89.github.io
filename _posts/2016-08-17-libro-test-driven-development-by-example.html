TDD simplifica el proceso de creación de código en tres pasos:

  • Crear un test que ponga a prueba el requerimiento
  • Escribe un código que pase el test
  • Refactoriza / Limpia tu código


En este libro se puede encontrar distintos métodos de realizar estos pasos, distintos patrones para cada una de las tres fases y buenas prácticas. Además de explicarlo todo a través de ejemplos paso a paso (en algunos casos pasos demasiado lentos).

Muy recomendado como introducción a TDD.
